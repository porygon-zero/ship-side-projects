# Correct Course

Use this protocol when new evidence materially invalidates value, scope, requirements, architecture, risk, readiness, or authority. Do not use it for routine implementation feedback.

1. Record the evidence and distinguish fact, inference, assumption, and unknown.
2. Identify the invalidated assumption, decision, criterion, or plan element.
3. Assess effects on user value, active scope, risk class, architecture, data, schedule, and authority.
4. Preserve completed work and evidence that remain valid; identify work to revise, discard, or isolate.
5. Recommend the smallest viable correction, alternatives when genuinely material, and any new user decision.
6. Update readiness and living records, then repeat only affected gates and acceptance evidence.

Use `assets/templates/course-correction.yaml` when the change is material enough to need a durable record.
